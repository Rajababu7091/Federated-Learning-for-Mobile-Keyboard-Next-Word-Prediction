COMMON UI ISSUES AND SOLUTIONS
================================================================================

ISSUE 1: "Model not loaded" badge in UI
CAUSE: Server started but model file doesn't exist
SOLUTION: 
  - Click "Train Model" button in the Train tab
  - Wait for training to complete (25 rounds)
  - Model will be saved and badge will turn green

ISSUE 2: Predictions not working
CAUSE: Model not trained yet
SOLUTION:
  - Check the model badge in the sidebar (should say "Model Ready")
  - If it says "Not Loaded", train the model first
  - After training, refresh the page

ISSUE 3: "Error: Model not trained yet" when clicking predict
CAUSE: API call made before model is loaded
SOLUTION:
  - Wait for server to fully start (check console for "[OK] Model loaded")
  - Refresh the browser page
  - Try prediction again

ISSUE 4: Blank page or CSS not loading
CAUSE: Static files not found
SOLUTION:
  - Verify folder structure:
    frontend/
      static/
        css/style.css
        js/app.js
      templates/
        index.html
  - Restart the server
  - Clear browser cache (Ctrl+Shift+R or Cmd+Shift+R)

ISSUE 5: CORS errors in browser console
CAUSE: Browser security restrictions
SOLUTION:
  - Make sure you're accessing http://localhost:5000 (not file://)
  - CORS is already enabled in app.py
  - Try a different browser

ISSUE 6: Port 5000 already in use
CAUSE: Another application using port 5000
SOLUTION:
  - Stop other applications using port 5000
  - Or change port in app.py:
    app.run(host='0.0.0.0', port=5001, debug=False)
  - Then access http://localhost:5001
